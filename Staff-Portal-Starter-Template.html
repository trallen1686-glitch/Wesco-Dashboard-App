ℹ How the links on this page work

Every tile and chip on this page opens a URL from a small configuration table at the top of the script — TAB_LINKS for the six pages everyone gets, QUICK_TOOLS for the form templates already built for the WE App. They all start blank, so clicking one before it's configured shows a reminder instead of a dead link. Add the real SharePoint folder, calendar, or screen URL for each one and it becomes a live link — nothing else on the page needs to change.

Wes Atkinson's page also gets three extra tiles that nobody else sees — Approvals (Estimates, Change Orders, Takeoffs & Material Sheets), Master To-Do List, and Master Urgent List — configured in PERSON_EXTRA_TABS, right below TAB_LINKS. Give someone else their own extra tabs the same way by adding another name key there. The Approvals tile is a good candidate to point at the Change Order template's own approval queue (Change_Order_Template.html?person=Wes Atkinson) once Estimates, Takeoffs, and Material Sheets have somewhere similar to send their approvals — right now it's one link, so it'll need to point at wherever all four end up living, whether that's one combined list or a page of its own.

Assigned Projects and To Do List are marked personAware: true, so once their base URL is filled in, this page automatically appends ?person=NAME to it — the same convention the To-Do widget already uses — so each person lands on their own filtered view instead of everyone's.

This page itself follows that same convention: opening it with ?person=Jesse Warren (for example) in the URL goes straight to that person's starter view, skipping the name picker. That's the URL each person's own screen in the WE App should embed this control with.

Estimator, PM, Superintendent, and Management staff are listed here. Any other office or admin staff aren't on the roster yet — add them to the PEOPLE list at the top of the script (same pattern as the others) if they need a starter page too.

Messages works right now — pick a name, type, hit Send, it shows up in that conversation with a chat bubble and an unread badge on the tile. One important limit to know before rolling it out: it saves to this browser's own localStorage, the same as everything else on this page. Two people on two different computers are each writing to their own separate copy, so they won't actually see each other's messages — a message "sent" here doesn't leave the device it was sent from. For that to work for real between staff, it needs either a shared backend (a SharePoint list or Dataverse table, polled or pushed to on an interval) behind loadMessages() / saveMessages(), or, more realistically, since Wesco already has Microsoft 365 — just point the Messages tile at Teams chat instead of building a custom inbox at all. Happy to wire up whichever direction makes more sense.

All Projects, Active Projects, and Pending Projects are three separate views into what's presumably one underlying project list, filtered by status — they're set up as plain company-wide links (not personAware) since "active" and "pending" read as company-wide states rather than something scoped to one person. Forms is a general company form library link; it's separate from the Quick Tools chips below, which link straight to the individual form templates already built.
